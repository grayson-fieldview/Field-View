Goal: replace the current "Upload Photos" button behavior with a modal 
that has a dashed-bordered drop zone. User can click the drop zone to 
open the local file picker, OR drag & drop image files directly onto 
the zone. Reference: CompanyCam's Add Photos modal (clean dashed border, 
centered instruction text, X close in top-right).

PHASE 2 — DESIGN PROPOSAL (before building, propose the structure)

Modal layout sketch (text only, no code yet):
- shadcn/ui Dialog wrapper, centered, max-w around lg or xl
- Header row: "Add Photos" left, X close right
- Main area: dashed border (2px dashed, neutral-300 light / neutral-600 
  dark, 12px corner radius), large padding, takes ~60vh
- Inside drop zone: centered icon (Upload from lucide) + text "Click 
  here or drag & drop photos"
- Below drop zone: thin terms-of-service-style text (skip if no TOS to 
  link to)
- Below that: when files are uploading, show a list of files with 
  progress bars or status indicators; modal stays open during upload

Behavior:
- Click anywhere in the dashed zone → triggers hidden <input type="file" 
  multiple accept="image/*" /> click()
- Drag file over modal → border highlights (e.g. dashed → solid + 
  primary color), background tint
- Drop files → validate (image types only), kick off upload, append to 
  in-progress list
- Multiple files supported (drop a folder's worth)
- During upload: drop zone stays interactive — user can drop more
- On close: if uploads in progress, confirm before close (or let them 
  continue in background — pick the existing pattern)
- On success: toast + invalidate the photos query so new pics appear 
  in the project

Mobile: touch devices don't fire dragover/drop the same way — the 
click-to-browse path is the primary fallback. Verify that the click 
path works on touch screens too (iPad / iPhone Safari for the web 
dashboard, even though primary mobile is the native app).
