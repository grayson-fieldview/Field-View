ADDITION TO ORIGINAL SPEC: hourly rate

Add to users table:
- hourly_rate_cents: integer NULL (cents, e.g. 2500 = $25.00/hr). Nullable means "not set."

Add to time_entries table:
- rate_cents_snapshot: integer NULL
  Rationale: "rate at time of work" is the payroll-correct model. On clock-out (and on manager manual-entry creation), copy the user's current hourly_rate_cents into this column. If the user's rate later changes, historical entries keep their original cost. Null means "user had no rate set when this entry was completed" — UI shows "—" for that row's cost.
  When PATCH updates an entry's clock_in/clock_out, do NOT touch rate_cents_snapshot — preserve the original payroll record. Manager can override by passing rateCentsSnapshot explicitly in the PATCH body if they need to correct it.

API endpoints — additions/changes:

- PATCH /api/users/:id  (manager-only — requireWriteAccess + requireAdminOrManager)
  Add hourlyRateCents to the accepted update fields if it doesn't already exist on this endpoint. If this endpoint doesn't exist, create it. Body: { hourlyRateCents?: number | null, timesheetEnabled?: boolean }.
  Validation: hourlyRateCents must be integer >= 0 or null. Reject decimals (Zod: z.number().int().nonnegative().nullable()).
  Account-scope: target user must be in same account as caller.

- GET /api/users — manager-only callers see hourlyRateCents and timesheetEnabled in the response. Standard/restricted callers do NOT see hourlyRateCents in any user object (including their own — strip it from the response). They MAY see timesheetEnabled on themselves only.
  Important: this is response-level filtering. Audit every existing endpoint that returns user objects and confirm hourlyRateCents is stripped for non-managers. List the endpoints you audited in your report-back.

- POST /api/timesheets/clock-out — when closing the entry, snapshot the user's current hourly_rate_cents into rate_cents_snapshot. Same for POST /api/timesheets manager-manual-create.

- POST /api/timesheets/clock-in — do NOT snapshot here. Snapshot happens at clock-out so the rate reflects the rate that was in effect when the work was completed.

- GET /api/timesheets — include rate_cents_snapshot in the response only for manager callers. Strip it for the user viewing their own entries.
