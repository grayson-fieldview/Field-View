## 3. Update payment method action

Implement handleUpdatePaymentMethod that opens the Stripe Customer 
Portal:

  const handleUpdatePaymentMethod = async () => {
    try {
      const res = await apiRequest("POST", "/api/create-portal-session", {});
      const data = await res.json();
      if (data.url) {
        window.location.href = data.url;
      } else {
        toast({ 
          title: "Error", 
          description: "Could not open billing portal", 
          variant: "destructive" 
        });
      }
    } catch (error: any) {
      toast({ 
        title: "Error", 
        description: error.message || "Could not open billing portal", 
        variant: "destructive" 
      });
    }
  };

The endpoint POST /api/create-portal-session already exists (used by 
Settings → BillingCard's "Manage Billing" button). Same pattern, same 
flow.

If the user is in read_only state but stripeCustomerId is somehow 
null (corrupt state), POST /api/create-portal-session will fail. 
Show the toast error gracefully — don't crash.
