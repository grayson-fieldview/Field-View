PART A — CSV EXPORT

New endpoint: GET /api/timesheets/export.csv
  Query params (same as GET /api/timesheets):
    - startDate (required, ISO)
    - endDate (required, ISO)
    - projectId (optional)
    - userId (optional, but ignored if not manager)
    - format: "generic" | "gusto" | "quickbooks" (default "generic")
  
  Auth: requireReadAccess + requireAdminOrManager (manager-only)
  Account-scoped same as GET /api/timesheets
  
  Response:
    - Content-Type: text/csv; charset=utf-8
    - Content-Disposition: attachment; filename="timesheets-{startDate}-to-{endDate}.csv"
    - Body: CSV string
  
  CSV escaping: any field containing comma, quote, or newline gets
  wrapped in double quotes; double quotes inside fields get doubled.
  Use a small inline helper or install csv-stringify (npm) — agent's
  call based on what's already in package.json.

  Generic format columns:
    User Name, Email, Project, Clock In (ISO), Clock Out (ISO),
    Duration (hours, decimal 2dp), Source, Hourly Rate (USD),
    Cost (USD), Notes, Edited By, Edited At
  
  Gusto format columns (per Gusto's CSV import for hours):
    Employee Email, First Name, Last Name, Work Date, Regular Hours,
    Hourly Rate, Notes
    - Aggregate per (user, project, work_date) — Gusto wants daily
      summary rows, not individual clock-ins
    - Work Date = YYYY-MM-DD in user's local tz (default US Eastern;
      flag this — see open question below)
  
  QuickBooks format columns (per QB Time/QuickBooks Online time entry):
    Date, Employee, Customer:Job, Service Item, Duration (HH:MM),
    Hourly Rate, Notes
    - Per-entry, not aggregated (QB accepts individual entries)
    - Customer:Job uses the project name as the customer identifier
    - Service Item: leave blank for v1 (user can map manually in QB)

  Active entries (clock_out IS NULL): EXCLUDE from export. Currently-
  clocked-in entries don't belong in payroll. Note this in the response
  somehow — perhaps a comment row at top: "# {N} active entries excluded"
  Actually scratch that — comment rows break some CSV importers. Just
  silently exclude. Document the behavior in the UI tooltip.
  
  Null-rate entries: include them, show empty Hourly Rate and empty
  Cost cells. Manager can fill in manually if needed before importing.

UI: Export button (or button group)
  - Place near the date range picker / project filter row
  - Either a single "Export CSV" button with a dropdown for format,
