## 1. Add sendInvitationEmail to server/services/email.ts

Mirror the existing email-sending functions in this file. Look at 
how sendPasswordResetEmail or sendEmailVerificationEmail are structured 
and follow the same pattern.

Function signature:

  export async function sendInvitationEmail(params: {
    to: string;                    // recipient email
    inviterName: string | null;    // who is inviting (full name, fallback to inviter email)
    inviterEmail: string;          // fallback for missing name
    accountName: string;           // "X's Team" usually
    role: string;                  // admin, manager, standard, restricted
    inviteUrl: string;             // the URL the recipient clicks
    expiresAt: Date;               // when the invite link expires
  }): Promise<{ success: boolean; error?: string }>

Subject line: 
  `${inviterName || inviterEmail} invited you to join ${accountName} on Field View`

Body content (HTML, mirror the styling of other Field View emails):
- Greeting: "Hi,"
- Lead text: "${inviterName || inviterEmail} invited you to join 
  ${accountName} on Field View as a ${roleLabel}."
- Brief description of what Field View is (1-2 sentences). Match the 
  tone of existing emails.
- Primary CTA button → inviteUrl. Use the existing brand orange 
  (#F09000) for the button background.
- "This invitation expires on ${formatted expiresAt}."
- Plain-text fallback: "Or paste this link into your browser: ${inviteUrl}"
- Standard footer (whatever the other emails use).

Where roleLabel maps:
- admin → "Admin"
- manager → "Manager"
- standard → "Standard User"
- restricted → "Restricted User"

Plain-text version: include the same content stripped of HTML for 
email clients that don't render HTML.

Logging: match the pattern of other email functions. console.log on 
success, console.error on failure.
