Task: Implement role/project-aware filtering for the task assignee picker.

CONTEXT
Currently GET /api/users (server/routes.ts:2270-2280) returns every user in the account regardless of role or project membership, and the task picker in client/src/components/task-form-dialog.tsx renders all of them. A restricted user who isn't a member of project X can currently be assigned a task on project X — they'll never see it. That's the bug.

DESIRED BEHAVIOR for the task assignee picker on a given project X:
- Show all admin, manager, and standard users in the account (regardless of project membership)
- Show restricted users ONLY if they are members of project X (via project_assignments)
- Behavior of /api/users for any other consumer must not change

APPROACH
Add an optional query param to /api/users:
- GET /api/users (no params) → unchanged, returns all users
- GET /api/users?assignableForProjectId=39 → returns the filtered list per the rule above
