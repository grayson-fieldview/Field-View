=== DATA MODEL ===
Create a new Drizzle schema for annotations table:

- id: uuid primary key
- media_id: foreign key to media table (cascade delete)
- user_id: foreign key to users table (who drew it)
- strokes: jsonb (array of stroke objects)
- created_at: timestamp default now
- updated_at: timestamp default now

Stroke JSON schema (each element in strokes array):
{
  "id": "stroke-uuid",
  "type": "pencil" | "arrow" | "rectangle" | "circle" | "line",
  "color": "#hex",
  "width": number,
  "points": [{"x": 0.42, "y": 0.18}, ...]  // normalized 0-1 coordinates so annotations scale with image
}

Use normalized 0-1 coordinates (x and y as fractions of image width/height). This makes annotations resolution-independent.

Create and run the migration. Use the existing Drizzle migration pattern in the repo.

=== API ENDPOINTS ===
All require authentication. User can only modify their own annotations, but can view annotations from anyone on photos they have access to.

GET /api/media/:mediaId/annotations
  Returns array of all annotation records for the media

POST /api/media/:mediaId/annotations
  Body: { strokes: [...] }
  Creates new annotation record owned by current user. Returns created record.

PUT /api/annotations/:annotationId
  Body: { strokes: [...] }
  Updates strokes. Only owner can update. Returns updated record.

DELETE /api/annotations/:annotationId
  Only owner can delete. Returns 204.

Add these to the existing route file that handles media endpoints. Match the existing error handling and auth middleware patterns.
