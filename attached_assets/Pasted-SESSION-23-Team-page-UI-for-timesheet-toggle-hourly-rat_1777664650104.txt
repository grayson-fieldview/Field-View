   - Label: "Timesheet tracking"
   - Switch component (use whatever the codebase already uses — likely
     shadcn/ui Switch)

   - Optimistic update with React Query, rollback on error

B. Hourly rate
   - Label: "Hourly rate"
   - Display:
     - If hourlyRateCents is null → "Not set" (muted text) with a "Set rate"
       link/button
     - If set → "$25.00/hr" (format from cents: cents / 100, two decimals)
       with an "Edit" link/button
   - Click opens a small modal (or inline editor — match whatever pattern
     the existing role-change uses)
   - Modal contents:
     - Numeric input, accepts decimals (e.g., "25.50")
     - Convert to cents on submit: Math.round(parseFloat(input) * 100)
     - Validation:
       - Must parse to a finite number >= 0
       - After conversion, must be a non-negative integer
       - Reject if input has more than 2 decimal places
       - Show "Enter a valid hourly rate, e.g. 25.50"
     - "Clear rate" button (sets to null)
     - "Save" button — PATCH /api/users/:userId with
       { hourlyRateCents: number | null }
   - On success: close modal, refresh user list, toast

  invent a new rule)
- If user's role is "owner" / "admin" and current user is "manager" (not
  admin): defer to existing permission patterns. Don't invent new rules —
