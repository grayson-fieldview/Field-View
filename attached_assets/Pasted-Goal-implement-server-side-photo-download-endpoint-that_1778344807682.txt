- Server endpoint contract (recap, you already speced this — just 
  confirm in plan):
  - POST /api/projects/:id/media/download
  - Body: { mediaIds: number[] }, zod schema: 
    z.object({ mediaIds: z.array(z.number().int().positive()).min(1).max(50) })
  - Auth: requireReadAccess + accountId tenancy check + restricted-
    role assignment check (mirror GET /api/projects/:id pattern)
  - Validation: filter mediaIds to ones that belong to the project, 
    drop unknowns silently (no info leak), 400 if intersection empty
  - Response headers: Content-Type: application/zip, 
    Content-Disposition: attachment with sanitized filename, 
    Cache-Control: no-store
  - Filename: `${projectSlug}-photos-${yyyy-mm-dd}.zip` where 
    projectSlug is project.name.replace(/[^a-z0-9]+/gi,"-").toLowerCase().slice(0,50)
  - Zip body: archiver("zip", { store: true }) — no compression, 
    JPEGs/MP4s don't compress
  - Entry naming: use media.originalName, sanitize against path 
    traversal (strip / \ and leading .), de-dupe collisions with 
    " (2)", " (3)" suffix before extension
  - Error handling: per-file S3 fetch failure → log, skip, append to 
    _DOWNLOAD_ERRORS.txt manifest at end. archiver-level error → 
    destroy response (TCP-level failure, not corrupted zip). Pre-flight 
    failures (auth, validation) → standard JSON 4xx.
