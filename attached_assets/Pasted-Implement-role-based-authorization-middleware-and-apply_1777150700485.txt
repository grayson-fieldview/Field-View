# Context

Field View has a 4-role permission model: admin, manager, standard, restricted.
Roles are stored as a varchar on the users table: shared/models/auth.ts:31.
Today, role checks are done inline in each route handler (e.g., 
"if (currentUser.role !== 'admin' && currentUser.role !== 'manager')"). 
This is repetitive and easy to get wrong. We're consolidating to reusable middleware.

# Task

## Step 1: Create middleware file

Create a new file: server/middleware/auth.ts

Export three middleware functions:

1. requireAdmin(req, res, next)
   - 401 if not authenticated
   - 403 with { error: 'forbidden', message: 'Admin role required' } if req.user.role !== 'admin'
   - Otherwise next()

2. requireAdminOrManager(req, res, next)
   - 401 if not authenticated
   - 403 with { error: 'forbidden', message: 'Admin or Manager role required' } 
     if req.user.role is not in ['admin', 'manager']
   - Otherwise next()

3. requireOwnerAdmin(req, res, next)
   - 401 if not authenticated
   - 403 if req.user.role !== 'admin'
   - 403 if req.user.id !== req.user.account.owner_id
   - Otherwise next()
   - NOTE: accounts.owner_id does not exist yet — it's added in a future migration.
     Implement this middleware now but mark it with a TODO comment that says:
     "TODO: requires accounts.owner_id column (Session 1 schema migration). 
     Currently always returns 403 for safety. Do not apply to any route until owner_id exists."
   - Do NOT apply this middleware to any route in this prompt. Just create it.

All three middleware should use the existing isAuthenticated check pattern already 
used elsewhere in the codebase (look at how server/routes.ts does it today and match the style).
