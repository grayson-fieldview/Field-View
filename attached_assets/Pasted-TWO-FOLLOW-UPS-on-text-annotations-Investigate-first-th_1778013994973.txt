Issue 1: CANNOT RE-EDIT TEXT AFTER PLACEMENT
The build output mentioned "handleTextNodeClick (eraser deletes / text tool 
re-edits)". User reports clicking on existing text with the text tool 
active does NOT re-open the inline input.

INVESTIGATE:
- Find handleTextNodeClick in photo-viewer.tsx — read its actual logic
- Trace: when text tool is active and user clicks an existing text node 
  in their own annotation, does this handler fire? Or is the click 
  intercepted upstream (canvas pointer handler swallowing it, 
  pointer-events conflict, conditional gate blocking it)?
- Likely culprits: 
  a) handleTextNodeClick exists but is only attached for the eraser case
  b) The text node's onClick is gated by isAnnotating but not by 
     annotationTool === "text"
  c) The HTML overlay for "mine" text has pointer-events:none in the 
     wrong mode
  d) The click bubbles to a canvas handler that calls preventDefault first

FIX:
- Single click on an existing text node (mine) while text tool is active 
  → re-opens inline input at that text's position, pre-filled with 
  current content, with the existing id preserved (so save updates 
  in-place, doesn't duplicate)
- On Enter/blur: replaces the original. On Escape: cancels (original 
  text remains unchanged).
- Eraser tool branch unchanged (single click deletes).
- Other tools (pencil/arrow/circle/etc): clicking text does nothing — 
  text shouldn't get in the way of drawing.
