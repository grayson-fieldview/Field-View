I4. New endpoint: GET /api/me/referral
    - Auth required (must be logged in)
    - Logic:
      1. Read current user (id, email, full_name, 
         rewardful_affiliate_id, rewardful_referral_url)
      2. If user.rewardful_affiliate_id AND 
         user.rewardful_referral_url both exist → return cached 
         values + fetch fresh stats from Rewardful
      3. Otherwise, find or create affiliate:
         a. Try findAffiliateByEmail(user.email) 
         b. If not found, createAffiliate({ email, first_name, 
            last_name }) — split user.full_name into first/last 
            (split on first space; if no space, put all into 
            first_name and use empty string for last_name)
         c. Save affiliate.id to users.rewardful_affiliate_id
         d. Save affiliate.links[0].url to 
            users.rewardful_referral_url
      4. Fetch stats:
         - GET /v1/affiliates/:id should return visitors, leads, 
           conversions counts and commission totals
      5. Return JSON:
         {
           referralUrl: <users.rewardful_referral_url>,
           referralCode: <link token from URL, e.g. extract ?via= 
             param>,
           stats: {
             visitors: <number>,
             leads: <number>,
             conversions: <number>,
             unpaidCommissionsCents: <number>,
             paidCommissionsCents: <number>
           }
         }
    - On Rewardful API error: return 503 with 
      { error: "Couldn't load referral info, try again shortly" }
    - Log all errors to Sentry with context
