REVISED STRATEGY

Hybrid approach:
- WEB requests: validate Origin against allowlist (browsers set this
  reliably and cannot be forged from JS)
- MOBILE requests: require a custom header X-FieldView-Client: mobile-1
  on state-changing requests. Browsers cannot set arbitrary X- headers
  cross-origin without CORS approval (which we don't grant). The
  presence of this header acts as a CSRF defense for mobile, since
  attacker browsers can't forge it.
- WEBHOOKS (/api/stripe/webhook): path-prefix bypass.
- UNAUTH endpoints (login, register, forgot-password, reset-password,
  resend-verification): path-prefix bypass.
