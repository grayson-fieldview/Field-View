# Context

Field View has a Restricted role: users in this role can only access projects
they personally created OR were explicitly assigned to via the project_assignments table.

GET /api/projects/:id (around line 109) correctly enforces this:
  - First checks project.accountId matches user's accountId
  - Then for restricted users: looks up project_assignments and verifies either 
    assignment exists OR project.createdById === user.id
  - Returns 403 if neither

PATCH /api/projects/:id (around line 154) and DELETE /api/projects/:id (around line 173) 
do NOT enforce this. They only check accountId. This means a Restricted user 
can edit or delete any project in their account by guessing the integer ID, 
even though they can't see those projects in their list.

This is a real security bug. We're closing it tonight.
