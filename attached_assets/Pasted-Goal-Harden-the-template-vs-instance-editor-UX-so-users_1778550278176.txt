PHASE 2 — IMPLEMENT

CHANGE 1: Visual differentiation on the template editor page 
(client/src/pages/checklist-template-edit.tsx)

Add a prominent banner above the editor body — a colored strip 
or badge that makes the surface unmistakable. Suggested treatment:
- A horizontal bar across the top of the editor content area
- Use the app's accent orange (#f09004 — the canonical Field View 
  orange) as a left border or background tint
- Icon (FileText or LayoutTemplate from lucide-react) + text 
  "Editing template: {template.title}"
- Subtext below: "Changes here update the template only. They do 
  not affect checklists already created from this template."

The banner must be visible without scrolling, distinct enough that 
a user mid-task glancing at the screen knows immediately they're 
in the template editor, not a project's checklist.

CHANGE 2: Specific AlertDialog text — name the surface

For every delete confirm in the template editor, the title and 
description must explicitly say "template":

Examples (adapt to the actual existing dialog code):
- Item delete: 
  Title: "Delete this template item?"
  Description: "This removes the item from the template. Checklists 
  already created from this template will not be affected."
- Section delete:
  Title: "Delete this template section?"
  Description: "Items in this section will move to the Untitled 
  section. Checklists already created from this template will not 
  be affected."
- Option delete (for multiple choice):
  Title: "Delete this option?"
  Description: "This removes the choice from the template. 
  Checklists already created from this template will not be 
  affected."

For every delete confirm in the per-project checklist editor 
(checklist-section-editor.tsx etc.), the dialog text should 
explicitly say "this checklist":
- Item delete:
  Title: "Delete this item?"
  Description: "This removes the item from this checklist only. 
  The template (if any) is not affected."
- Section delete:
  Title: "Delete this section?"
  Description: "Items in this section will move to Untitled. 
  This only affects this checklist."

Same template-removal-disclaimer phrasing for the templates tab 
card delete on /checklists (already exists per Phase 2 — confirm 
the text matches the new pattern).

CHANGE 3: Cache invalidation audit + fix

For every mutation in the template editor, the invalidation must 
target ONLY template query keys (e.g. 
["/api/checklist-templates/:id", ...], 
["/api/checklist-templates/:id/items"], etc.). It must NOT 
invalidate any /api/checklists or /api/checklist-items keys.

For every mutation in the per-project checklist editor, the 
invalidation must target ONLY instance query keys 
(/api/checklists/:id/items, etc.) and NOT template keys.

If you find any mutation that cross-invalidates 
(template mutation touching instance cache or vice versa), fix it 
so each editor's mutations only invalidate its own surface. Paste 
before/after for any cache key you change.
