- On commit (Enter/blur), the saved shape persists with the current

ISSUE B (FEATURE): Drag-to-reposition text annotations
CompanyCam pattern — click + drag on a text node moves it. Was punted in 
v1; adding now.

DESIGN:
- Pointer down on a mine-text node + pointer moves > 5px before pointer 
  up = DRAG. Pointer down with no movement (or < 5px) = CLICK (existing 
  behavior — opens edit input).
- During drag: text follows the pointer, x/y normalized coords (relative 
  to imageRectStyle bounds, NOT viewport) update live.
- On release: new position committed to the annotation in annotations[] 
  buffer. If the drag started on a saved-mine text, promote to 
  edit-mine first (load row into annotations buffer) and apply the new 
  position — single motion, no separate "open editor first" step.
- Cursor: when hovering my text in interactive mode → cursor: move. 
  When dragging → cursor: grabbing.
- Use pointer events (onPointerDown / Move / Up) so this works on 
  touch screens too.
- Drag must not trigger a click-to-edit when released (suppress the 
  click handler if a drag was detected).
