B. Decimal validation — use regex on the raw input string, not float math
   Float-comparison approaches like Math.round(v*100) === v*100 are fragile
   (e.g., 0.1 + 0.2 issues). Use:
   
   const RATE_REGEX = /^\d+(\.\d{1,2})?$/;
   
   In dollarsToCents(input), trim the input first, then if !RATE_REGEX.test(trimmed)
   return null. Otherwise parseFloat and Math.round(value * 100). This rejects:
   - "25.999" (3 decimals)
   - "abc" (non-numeric)
   - "-5" (negative — leading minus fails the regex)
   - "" (empty)
   - " 25 " gets handled by trim
   
   Returns integer cents on valid input, null on invalid.

Currency formatting: plain "$25.00/hr" — do NOT use Intl.NumberFormat. US-only
product for now.
