═════════════════════════════════════════════════════════════

B1. Page header:
    - "← Back to Checklists" link → /checklists?tab=templates
    - Editable template title (inline rename, PATCH 
      /api/checklist-templates/:id on blur)
    - Optional description field (inline edit, same PATCH)

B2. Body — stack of section editors:
    - Items with NULL section_id render in an implicit "Untitled" 
      section at top (matches per-project pattern)
    - Below all sections: "+ Add Section" full-width button

B3. Data fetching:
    - useQuery: GET /api/checklist-templates/:id (template metadata)
    - useQuery: GET /api/checklist-templates/:id/sections
    - useQuery: GET /api/checklist-templates/:id/items
    - For multiple_choice items: lazy-fetch options via 
      /api/checklist-template-items/:itemId/options on first expand 
      (mirror the MultipleChoiceField pattern in 
      checklist-field-input.tsx)

B4. Section row component (new — likely 
    client/src/components/template-section-editor.tsx):
    - Drag handle (left edge)
    - Editable section title (inline, PATCH 
      /api/checklist-template-sections/:id on blur)
    - Renders the item rows inside
    - "+ Add item" footer with field type dropdown (default Yes/No); 
      on click creates POST /api/checklist-templates/:id/items with 
      { label: "Untitled item", fieldType, sectionId }
    - Delete section button (items in that section get section_id 
      set to NULL via ON DELETE SET NULL — they re-render in the 
      Untitled section, matching the per-project editor)

B5. Item row component (new — likely 
    client/src/components/template-item-editor.tsx):
    - Drag handle
    - Editable label (PATCH /api/checklist-template-items/:id on blur)
    - Field type dropdown: Yes/No, Rating, Text, Multiple Choice 
      (PATCH fieldType on change)
    - For multiple_choice ONLY: inline options editor
        - List of existing options with edit + delete inline
        - "+ Add option" input → POST 
          /api/checklist-template-items/:id/options with { label }
        - Drag-to-reorder options → POST 
          /api/checklist-template-items/:id/options/reorder
    - "Photo required to complete" toggle (PATCH photosRequired)
    - "Add notes" affordance — collapsible textarea (PATCH notes)
    - Delete button (one-click for items is fine; section delete 
      gets a confirm)

B6. Drag-to-reorder:
    - Match whatever drag library the per-project checklist editor 
      uses (check checklist-section-editor.tsx imports). If it uses 
      dnd-kit, use dnd-kit. If react-beautiful-dnd, use that. 
      Consistency matters.
    - Within section: POST /api/checklist-templates/:id/items/reorder 
      with orderedIds
    - Across sections: PATCH item with new sectionId, then reorder 
      destination
    - Section reorder: POST 
      /api/checklist-templates/:id/sections/reorder
