2. Overlap checking
   - Currently, the partial unique index prevents a user from having
     two ACTIVE entries (clock_out IS NULL) at once. But there's no
     constraint preventing manager-created OVERLAPPING completed entries
     (e.g., entry A: 8am-12pm, entry B: 11am-2pm — both valid by current
     index, conceptually nonsensical).
   - Question for your recommendation:
     a. Should POST and PATCH reject overlap with existing entries for
        the same user?
     b. What's the cleanest implementation? Likely a server-side check
        at write time: "SELECT count(*) FROM time_entries WHERE user_id
        = $1 AND id != $2 AND tstzrange(clock_in, COALESCE(clock_out,
        'infinity')) && tstzrange($3, $4)". The && operator requires
        the btree_gist extension OR we use boolean logic on the
        timestamps directly.
   - Recommend an approach. My instinct is YES on overlap rejection,
     using boolean timestamp logic (no new extension needed).
     Specifically: "WHERE user_id = $u AND id != $self AND clock_in 
     $newEnd AND COALESCE(clock_out, 'infinity'::timestamptz) > $newStart"
   - Edge cases to think through:
     a. Boundary touch: A ends at 12:00:00, B starts at 12:00:00. Allow.
     b. Sub-second overlap: A ends 12:00:00.500, B starts 12:00:00.499.
        Reject with clear error.
     c. Active entry vs new completed: if a user has an active entry
        (no clock_out) at 9am, and manager tries to add a completed
        entry from 8-10am, this overlaps. Reject.
   - Confirm the "boundary touch is allowed" rule is reasonable for
     contractors (yes — back-to-back jobs are common).

PART B — UI: Manual entry modal

Trigger:
  - "Add entry" button (with PlusIcon from lucide) in the filter bar,
    placed left of the export dropdown. Manager-only (already gated
    by the page's role redirect).

Modal contents:
  - Title: "Add timesheet entry"
  - User select: dropdown of users where timesheet_enabled = true in
    the account. Show name + (rate if set, otherwise "no rate").
    Required.
  - Project select: dropdown of all projects in account. Show name +
    address. Required.
  - Clock in: <input type="datetime-local"> required
  - Clock out: <input type="datetime-local"> required
  - Notes: <textarea> optional
  - Cancel / Save buttons

Validation (client-side):
  - All required fields present
  - clock_out > clock_in (parsed as Date objects)
  - If invalid, show inline error on the offending field, disable Save
  - On submit: convert datetime-local strings to ISO via
    new Date(localStr).toISOString() — this respects browser tz

Submit:
  - POST /api/timesheets with the body
  - On 200: close modal, refresh the entries list, toast "Entry added"
  - On 409 overlap: show inline error "This entry overlaps with an
    existing entry for [user name]." Keep the modal open so manager
    can adjust.
  - On 400 / 403 / 500: toast with the server's error message
  - On any error: do NOT clear the form
