CONFIRMED FINAL SPEC

Allowlist hosts (web Origin/Referer):
- app.field-view.com (exact)
- *.vercel.app (suffix — covers field-view.vercel.app + previews)
- *.replit.dev (suffix — Replit dev random subdomains)
- localhost (for future local dev)

Mobile signal: X-FieldView-Client: mobile-1 header on non-GET requests.

Path bypass list (no CSRF check at all):
- /api/stripe/webhook (Stripe-signature verified instead)
- /api/login
- /api/register
- /api/forgot-password
- /api/reset-password
- /api/resend-verification
- /api/logout (chicken-and-egg if session is broken)

CSRF_MODE env var:
- "warn" → run check, log [csrf] would-block on failure, but call
  next() (don't block)
- unset or "enforce" → return 403 on failure
- "off" → skip middleware entirely (killswitch for incidents)

PHASE 1 — IMPLEMENT (DO NOT DEPLOY YET)

WEB SERVER

1. Create server/middleware/csrf.ts (~70 LOC). Pseudocode from diagnose
   v2 is the right structure. Specifically:

   - Export csrfGuard: RequestHandler
   - SAFE_METHODS = GET, HEAD, OPTIONS → next()
   - PATH_BYPASS list → if req.path matches exactly OR startsWith path+"/"
     → next() (the +"/" check matters for paths like /api/stripe/webhook
     followed by anything, though Stripe is exact)
   - Mobile branch: if x-fieldview-client header equals "mobile-1" → next()
     - Note: Express normalizes headers to lowercase. Use req.headers["x-fieldview-client"]
   - Web branch: extract Origin (preferred) or Referer (fallback). Parse
     URL. Check host against allowlist (exact match for app.field-view.com
     and localhost; suffix match for the wildcards via host === suffix
     OR host.endsWith("." + suffix))
   - On failure: check CSRF_MODE
     - "warn" → console.warn with reason and basic request context
       (method, path, origin/referer, x-fieldview-client value if any)
       and call next()
     - "off" → next() (killswitch — no logging needed beyond startup)
     - default/enforce → return 403 with { error: "csrf",
       message: "CSRF check failed" }

2. On startup (server/index.ts boot logging), log the active CSRF mode
   once: console.log(`[csrf] mode=${process.env.CSRF_MODE || "enforce"}`).
   Helps visibility in production.

3. Mount the middleware in server/replit_integrations/auth/replitAuth.ts
   setupAuth(), AFTER passport.session() and BEFORE registerRoutes /
   the request logger. Or, if cleaner, mount it in server/index.ts
   between the express.json/urlencoded parsers and registerRoutes.
   Pick whichever insertion point requires fewer code changes and
   document the choice in the file.

4. Update replit.md with a short paragraph documenting:
   - What the CSRF strategy is (Origin allowlist for web + custom
     header for mobile)
   - The bypass paths and why
   - How to flip CSRF_MODE for warn-only / killswitch
   - The hard rule: if cors() middleware is ever added later, it MUST
     NOT allow X-FieldView-Client cross-origin or the mobile defense
     is bypassed. This is critical and easy to miss.
