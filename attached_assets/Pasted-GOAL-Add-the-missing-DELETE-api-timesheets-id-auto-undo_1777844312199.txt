Middleware: isAuthenticated (NOT requireWriteAccess, NOT requireAdminOrManager — any signed-in user can undo their own entry)

SAFETY CONSTRAINTS (all must pass; any failure → reject with appropriate status)

A. Entry must exist. If not found → 404. Do not leak existence beyond that.

B. Entry must belong to the authenticated user.
   - If entry.user_id !== req.user.id → return 404 (NOT 403). Same-shape response as "not found." This is intentional: we don't reveal that an entry exists for someone else.
   - This is the cross-account / cross-user privacy guarantee.

C. Entry must have source === 'auto_geofence'.
   - If source is 'manual' or 'edited' → return 400 with error message "Only auto-geofence entries can be auto-undone."
   - This prevents manual entries from being silently destroyed.

D. Entry must be within the 60-minute window from its clock_in timestamp.
   - If (NOW() - entry.clock_in) > 60 minutes → return 400 with error message "Auto-undo window has expired."
   - Window is measured from clock_in, not created_at, not the current time of an active session.

E. If all constraints pass → hard DELETE the row. Return 204 No Content (no body).
