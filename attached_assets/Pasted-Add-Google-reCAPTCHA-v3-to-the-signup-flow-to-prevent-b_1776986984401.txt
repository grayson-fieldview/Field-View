Add Google reCAPTCHA v3 to the signup flow to prevent bot signups that burn Stripe trial credits.

=== DESIGN DECISIONS ===
- reCAPTCHA v3 (invisible, score-based: 0.0 = bot, 1.0 = human)
- Apply ONLY to POST /api/register — not login, not password reset (rate limiting handles those)
- Score threshold: 0.5 (Google's recommended default — rejects obvious bots, allows unsure cases)
- Action tag: "signup" (for analytics in reCAPTCHA admin console)
- Fail-closed: if verification API is unreachable or returns invalid response, BLOCK the signup (safer default than fail-open)
- Dev skip: bypass reCAPTCHA entirely when NODE_ENV === "development" for local testing

=== ENV VARS (already set in Vercel) ===
- VITE_RECAPTCHA_SITE_KEY (public, frontend, available at build time via import.meta.env)
- RECAPTCHA_SECRET_KEY (private, backend only, via process.env)

- Set scriptProps={{ async: true, defer: true, appendTo: "head" }} so the reCAPTCHA script loads performantly and doesn't block page render

=== STEP 4: Backend verification ===

Create server/services/recaptcha.ts:

interface RecaptchaVerifyResult {
  success: boolean;
  score?: number;
  action?: string;
  hostname?: string;
  errorCodes?: string[];
}

export async function verifyRecaptchaToken(token: string, expectedAction: string): Promise<{ valid: boolean; reason?: string; score?: number }> {
  const secret = process.env.RECAPTCHA_SECRET_KEY;
  
  if (process.env.NODE_ENV === "development") {
    console.log("[recaptcha] dev mode — bypassing verification");
    return { valid: true, score: 1.0 };
  }
  
  if (!secret) {
    console.error("[recaptcha] RECAPTCHA_SECRET_KEY not set — failing closed");
    return { valid: false, reason: "misconfigured" };
  }

  if (!token) {
    return { valid: false, reason: "missing_token" };
  }

  try {
    const response = await fetch("https://www.google.com/recaptcha/api/siteverify", {
      method: "POST",
      headers: { "Content-Type": "application/x-www-form-urlencoded" },
      body: new URLSearchParams({ secret, response: token }).toString(),
    });

    if (!response.ok) {
      console.error("[recaptcha] verify API HTTP error:", response.status);
      return { valid: false, reason: "api_error" };
    }

    const data = await response.json() as RecaptchaVerifyResult;

    if (!data.success) {
      console.warn("[recaptcha] verification failed:", data.errorCodes);
      return { valid: false, reason: "invalid_token", score: data.score };
    }

    if (data.action !== expectedAction) {
      console.warn(`[recaptcha] action mismatch — expected ${expectedAction}, got ${data.action}`);
      return { valid: false, reason: "action_mismatch", score: data.score };
    }

    const SCORE_THRESHOLD = 0.5;
    if ((data.score ?? 0) < SCORE_THRESHOLD) {
      console.warn(`[recaptcha] low score: ${data.score} (threshold: ${SCORE_THRESHOLD})`);
      return { valid: false, reason: "low_score", score: data.score };
    }

    console.log(`[recaptcha] verified, score: ${data.score}`);
    return { valid: true, score: data.score };
  } catch (err) {
    console.error("[recaptcha] verification threw:", err);
    return { valid: false, reason: "exception" };
  }
}

=== STEP 5: Gate the register endpoint ===

In server/replit_integrations/auth/replitAuth.ts, modify the POST /api/register handler:

1. Import verifyRecaptchaToken at top of file.

2. At the VERY START of the handler (before all other validation), extract recaptchaToken from req.body and verify it:

const { recaptchaToken } = req.body;
const recaptchaResult = await verifyRecaptchaToken(recaptchaToken, "signup");
if (!recaptchaResult.valid) {
  console.warn(`[register] reCAPTCHA failed for ${req.body.email}: ${recaptchaResult.reason}, score: ${recaptchaResult.score}`);
  return res.status(403).json({ 
    error: "security_check_failed", 
    message: "Security verification failed. Please try again." 
  });
}

3. Continue with all existing validation (email format, password length, duplicate check, etc.).
