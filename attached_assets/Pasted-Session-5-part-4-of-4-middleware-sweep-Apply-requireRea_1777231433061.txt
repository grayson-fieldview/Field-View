# Context

We have three subscription-gating middlewares:

- requireActiveSubscription (legacy, in 
  server/replit_integrations/auth/replitAuth.ts) — allows "active" 
  and "trialing" plus "trial" with non-expired trialEndsAt. Returns 
  403 otherwise.
- requireReadAccess (new) — allows "full" + "read_only" access levels. 
  Returns 402.
- requireWriteAccess (new) — allows "full" access level only. Returns 402.

Today, every authenticated endpoint uses requireActiveSubscription. 
We're swapping:
- GET endpoints → requireReadAccess (expand read access to lapsed-grace users)
- POST/PATCH/DELETE endpoints → requireWriteAccess (enforce read-only mode)

# Task

## 1. Find all endpoints using requireActiveSubscription

In server/routes.ts, search for every line containing 
"requireActiveSubscription". List each one with file:line and HTTP 
method (GET/POST/PATCH/DELETE).

## 2. Apply the swap

For each endpoint:
- If method is GET → swap requireActiveSubscription for requireReadAccess
- If method is POST, PATCH, PUT, or DELETE → swap requireActiveSubscription 
  for requireWriteAccess

Two specific endpoints get extra attention:

a. POST /api/account/seats — currently uses requireAdminOrManager only. 
   ADD requireWriteAccess to this endpoint (chain with requireAdminOrManager). 
   A read-only user should not be able to add seats.

b. POST /api/create-portal-session — locked-out users need to access 
   the Stripe Customer Portal to pay. Do NOT add requireWriteAccess. 
   Whatever middleware it has today, leave it alone. If it currently 
   has requireActiveSubscription, REMOVE it (don't replace with 
   requireWriteAccess) so locked users can still access the portal.
