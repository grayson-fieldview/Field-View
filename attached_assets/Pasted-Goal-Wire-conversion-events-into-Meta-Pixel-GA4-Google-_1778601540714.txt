    b) trial_started — find where the user lands on /dashboard for

    c) card_added — fires when Stripe webhook updates the user with 
       a stripeCustomerId. This is the canonical "Add Payment Info" 
       conversion. Two options:
       i)  Client-side: on the /subscribe?checkout=success callback 
           handler in subscribe.tsx — fires when Stripe redirects 
           back after successful checkout

           customer.subscription.created or checkout.session.completed

       success callback handler in subscribe.tsx and confirm it 
       exists.

       active (trial converted to paid, or paid from day 1). Two

           this is server-side only (webhook 
           invoice.paid or customer.subscription.updated with 
           status='active' and trial ending)
       
       Per memory, Field View runs a 14-day trial, so (ii) is the 
       correct interpretation for "real subscription started." 
       But the client browser won't be open when the trial ends 
       in most cases, so this event has to fire server-side and 
       be relayed via Conversions API (Meta) and the GA4 
       Measurement Protocol. FLAG this complexity — it's a real 
       infrastructure piece.
