IMPLEMENTATION (after I approve)

Schema additions:

1. New table `time_entries`:
   - id: uuid PK, default gen_random_uuid()
   - account_id: uuid NOT NULL, FK -> accounts.id
   - user_id: uuid NOT NULL, FK -> users.id
   - project_id: uuid NOT NULL, FK -> projects.id
   - clock_in: timestamptz NOT NULL
   - clock_out: timestamptz NULL  (null = active entry)
   - source: enum 'manual' | 'auto_geofence' | 'edited', default 'manual'
   - notes: text NULL
   - edited_by_user_id: uuid NULL, FK -> users.id
   - edited_at: timestamptz NULL
   - original_clock_in: timestamptz NULL  (audit: pre-edit value)
   - original_clock_out: timestamptz NULL (audit: pre-edit value)
   - created_at, updated_at: timestamptz NOT NULL default now()

   Indexes:
   - (account_id, user_id, clock_in DESC)  — primary query: user's entries by date
   - (account_id, project_id, clock_in DESC) — manager view by project
   - Partial unique index on (user_id) WHERE clock_out IS NULL — enforces "max one active entry per user" at the DB level

2. Add to `users` table:
   - timesheet_enabled: boolean NOT NULL default false

API endpoints (all require auth, all account-scoped):

- POST /api/timesheets/clock-in
  Body: { projectId: uuid, notes?: string }
  Validates: user.timesheet_enabled = true (else 403 "Timesheet tracking not enabled for this user")
  Validates: project belongs to user's account
  Validates: no existing active entry for user (DB constraint backs this up; return 409 if violated)
  Returns: created entry

- POST /api/timesheets/clock-out
  Body: { notes?: string }
  Closes user's active entry. 404 if no active entry. Appends notes if provided.
  Returns: updated entry

- GET /api/timesheets/active
  Returns the user's currently-open entry, or null.

- GET /api/timesheets?startDate=&endDate=&userId=&projectId=
  Date range required. Other filters optional.
  Regular users: forced to userId = self regardless of query param.
  Managers (owner/admin): can query any user in their account; if no userId, returns all in account.
  Returns array sorted by clock_in DESC.

- POST /api/timesheets   (manager-only)
  Body: { userId, projectId, clockIn, clockOut, notes? }
  Manual entry by manager. Validates clock_out > clock_in. source = 'manual'.

- PATCH /api/timesheets/:id   (manager-only)
  On first edit of any entry, copy current clock_in/clock_out into original_clock_in/original_clock_out.
  Set edited_by_user_id = current user, edited_at = now(), source = 'edited' (if it wasn't already).
  Allow updating: clock_in, clock_out, notes, project_id.

- DELETE /api/timesheets/:id   (manager-only)
  Hard delete for now. We can soft-delete in a later pass if needed.

Validation rules across all write endpoints:
- clock_out > clock_in (when both present)
- All FKs scoped to authenticated user's account
- Times accepted as ISO strings, stored as timestamptz
