# Fix 1: Remove cookie/session content from request logging

server/replit_integrations/auth/replitAuth.ts:144-169 has an [auth-diag] 
console.log block that runs on every /api/* request and logs:
- rawCookieHeader (full cookie header including signed session cookie)
- cookieFirstChars (first 50 chars of cookie)
- sessionID
- sessionPassport (which includes user.id)

If these logs are forwarded to any third-party log aggregator (Sentry, 
Vercel log drains, CloudWatch, etc.), session cookies are leaked and 
session hijacking becomes trivial for anyone with log read access. 
This defeats httpOnly.

## Task

Remove the [auth-diag] block entirely. We don't need ongoing diagnostic 
logging on every request — if we ever need to debug a specific request, 
we'll add targeted logs at that time.

If there's any reason to keep some diagnostic info, the only safe fields 
are: { path, method, isAuthenticated: !!req.user, hasUser: !!req.user }. 
Nothing involving the cookie, sessionID, or session passport contents.

# Fix 2: Delete /api/admin/setup-account endpoint

server/routes.ts (around line 1135-1150) defines POST /api/admin/setup-account.

Problems:
- Uses SESSION_SECRET as a shared "setupKey" — secret reuse, compromise of 
  one compromises both
- No rate limiter — brute-forceable
- Functionally redundant with bootstrapAdminAndOrphans in server/index.ts:113 
  which auto-provisions admins on every server start

## Task

Delete the entire endpoint. No replacement needed — the bootstrap function 
covers the use case.
