## 2. GET /api/tasks (around line 314-322 in server/routes.ts)

This endpoint returns all tasks in the account. Today it has no 
Restricted-user filter, meaning Restricted users see tasks for projects 
they can't access.

The fix is more involved than the previous one because it's a list 
endpoint, not a single-item lookup. The pattern from GET /api/projects 
(around line 88-107) applies:

After fetching all tasks for the account, if req.user.role === "restricted",
filter the result set to only tasks whose projectId is one of the projects 
the user has access to (project they created OR project they're assigned to).

Pseudocode:
  let allTasks = await storage.getAllTasks(accountId);
  if (req.user.role === "restricted") {
    // Get all projects the restricted user can see
    const accessibleProjects = await db.select({ id: projects.id })
      .from(projects)
      .leftJoin(projectAssignments, and(
        eq(projectAssignments.projectId, projects.id),
        eq(projectAssignments.userId, req.user.id)
      ))
      .where(and(
        eq(projects.accountId, req.user.accountId),
        or(
          eq(projects.createdById, req.user.id),
          isNotNull(projectAssignments.projectId)
        )
      ));
    const accessibleIds = new Set(accessibleProjects.map(p => p.id));
    allTasks = allTasks.filter(t => accessibleIds.has(t.projectId));
  }
  res.json(allTasks);

Match the existing import patterns in the file. If or/isNotNull aren't 
already imported from drizzle-orm, add them.

If this is too big a change for the storage layer, just do the filter 
in-memory on the existing allTasks result by checking each task's 
projectId. Less efficient but simpler.
