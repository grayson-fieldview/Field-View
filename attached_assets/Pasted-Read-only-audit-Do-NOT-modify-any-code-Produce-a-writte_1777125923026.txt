## The intended permission model

There are 4 roles. This is the source of truth I want the code audited against:

- **Admin** — Has complete control over the account. Can do everything, including billing, account settings, user management, and access to every project. Admin is the ONLY role that can touch billing.
- **Manager** — Access to all projects AND can manage users (invite, change roles, remove). Cannot access billing or account-level destructive actions.
- **Standard** — Access to all projects, but CANNOT manage users. Read/write on projects, but no user invite/remove/role-change ability.
- **Restricted** — Can only access projects they personally created OR were explicitly assigned to via project_assignments. Cannot see other projects in the account at all.
