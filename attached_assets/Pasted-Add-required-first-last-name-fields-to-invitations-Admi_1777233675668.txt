# Context

Today, Admins invite users by entering email + role. We're adding 
required first/last name fields:
- Personalize the email greeting ("Hi Sarah,")
- Pre-fill the registration form

Names are REQUIRED on the invite form. Existing pending invitations 
(created before this change) will have NULL names — that's fine, 
they're grandfathered in. New invitations require names.

  lastName: text("last_name"),

don't break. Required-ness is enforced at the application layer

After schema edit, run `npm run db:push` to apply against the dev DB.

## 2. Backend: POST /api/invitations validation

Accept firstName and lastName in the request body. Validate they're 
non-empty strings:

  const { email, role, firstName, lastName } = req.body || {};
  // Existing email + role validation stays
  // Trim and require non-empty:
  const trimmedFirst = (typeof firstName === "string" ? firstName.trim() : "");
  const trimmedLast = (typeof lastName === "string" ? lastName.trim() : "");
  if (!trimmedFirst || !trimmedLast) {
    return res.status(400).json({ message: "First and last name are required." });
  }

Store the trimmed values on the new invitation row.
